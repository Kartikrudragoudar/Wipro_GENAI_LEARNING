Create a full-stack project named medilens using a LOOP-Engineering architecture.

Product name:
MediLens - AI Medical Report Summarizer

Product concept:
Build a healthcare-focused AI document assistant that helps users upload medical lab reports, extract key lab values, compare them with reference ranges, explain the results in simple language, and generate questions to ask a doctor.

This product must be designed as a safe healthcare decision-support tool, not a diagnosis tool.

Important safety positioning:
MediLens does not diagnose disease, prescribe treatment, replace doctors, or provide emergency medical advice.
It helps users understand report terminology, organize lab values, and prepare better questions for qualified healthcare professionals.

Core LOOP-Engineering workflow:
Design the whole product around this loop:

Upload → Extract → Structure → Compare → Explain → Question → Export → Review

Each loop stage must be visible in the UI and represented in the backend architecture.

Loop stages:

1. Upload Loop
- User uploads a PDF or image medical report.
- System validates file type and file size.
- System creates a report session.
- UI shows upload status and file metadata.

2. Extraction Loop
- Backend extracts text from the uploaded report.
- For PDFs, use pdfplumber or a similar Python PDF text extraction library.
- For images, use OCR such as Tesseract or a mock OCR service for MVP.
- UI shows extracted text in a review panel.
- If extraction quality is low, user can manually edit extracted text.

3. Structuring Loop
- Backend detects common lab values from extracted text.
- Detect fields such as:
  - Hemoglobin
  - WBC
  - RBC
  - Platelets
  - Glucose
  - HbA1c
  - Cholesterol
  - HDL
  - LDL
  - Triglycerides
  - Creatinine
  - Urea
  - Vitamin D
  - TSH
  - ALT
  - AST
- Store each detected value with:
  - test_name
  - value
  - unit
  - reference_range
  - source_text
  - confidence_score

4. Comparison Loop
- Backend compares extracted values with reference ranges.
- Mark each value as:
  - normal
  - low
  - high
  - needs_review
  - unknown_range
- UI highlights values clearly but calmly.
- Avoid alarming language.
- Use neutral labels like “Outside reference range” instead of “danger.”

5. Explanation Loop
- Backend generates patient-friendly explanations.
- Use a provider-based AI architecture.
- Start with a MockAIProvider so the app works without API keys.
- The explanation must be simple, cautious, and non-diagnostic.
- Each explanation should include:
  - what the test commonly measures
  - whether the value is within reference range
  - why a doctor may review it
  - what context may affect interpretation

6. Doctor Questions Loop
- Backend generates suggested questions users can ask a doctor.
- Questions must be practical and safe.
- Example:
  - “Can you explain what this result means in my overall health context?”
  - “Should this test be repeated?”
  - “Are there lifestyle or medication factors that may affect this value?”

7. Export Loop
- User can export a clean summary as PDF.
- Export should include:
  - uploaded report name
  - extracted values table
  - highlighted out-of-range values
  - simple-language summary
  - doctor questions
  - safety disclaimer

8. Review Loop
- User can view report history for the current browser/session.
- User can reopen a previous report analysis.
- For MVP, store report history locally.
- Structure the app so database storage can be added later.

- run_comparison_loop()
- run_explanation_loop()
