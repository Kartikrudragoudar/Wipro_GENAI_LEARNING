Create a full-stack project named ecopulse using a LOOP-Engineering architecture.

Product name:
EcoPulse - Smart Energy Usage Optimizer

Product concept:
Build a ClimateTech energy analytics dashboard that helps households and small businesses upload electricity usage data, detect peak consumption patterns, estimate cost and carbon emissions, generate practical energy-saving recommendations, track actions, and improve usage over time.

This should not be just a static dashboard. It must be designed as a repeated improvement loop.

Core LOOP workflow:
Upload → Clean → Analyze → Estimate → Recommend → Act → Track → Improve

Each loop stage must be visible in the UI and represented in the backend architecture.

Main user journey:
1. User uploads an electricity usage CSV or loads a sample dataset.
2. System validates and cleans the data.
3. System analyzes energy usage patterns.
4. System estimates electricity cost and carbon emissions.
5. System generates personalized savings recommendations.
6. User selects actions to follow.
7. System tracks goal progress and completed actions.
8. System generates a next-cycle improvement plan.

LOOP stages:

1. Upload Loop
- User uploads electricity usage CSV or selects sample data.
- System validates file type and size.
- System checks required columns.
- System creates an energy session.
- UI shows upload status and file metadata.

CSV formats to support:
- timestamp,kwh
- date,kwh
- date,time,kwh
- month,kwh

If the uploaded CSV is missing required data, show a helpful error and offer a sample CSV.

2. Clean Loop
- Backend cleans the data using Pandas.
- Remove duplicate rows.
- Handle missing values.
- Normalize timestamp/date columns.
- Convert kWh values to numeric.
- Detect hourly, daily, or monthly data format.
- Flag invalid rows.
- Calculate data quality score.

UI should show:
- total rows
- valid rows
- ignored rows
- detected date range
- detected frequency
- data quality score

3. Analyze Loop
- Backend analyzes usage patterns.
- Calculate total kWh.
- Calculate average daily usage.
- Detect peak usage hour if hourly data exists.
- Detect peak usage day.
- Compare weekday vs weekend usage.
- Detect unusual usage spikes.
- Build monthly or daily trend data.

UI should show:
- total usage card
- average daily usage card
- peak usage card
- usage trend chart
- weekday vs weekend chart
- spike/anomaly list

4. Estimate Loop
- Backend estimates cost and emissions.
- User can enter tariff rate.
- User can enter or select grid emission factor.
- Provide default emission factor if user does not enter one.
- Calculate estimated cost.
- Calculate estimated CO2 emissions.
- Calculate cost per day.
- Calculate emissions per day.
- Compare with previous session if available.

Formula examples:
estimated_cost = total_kwh * tariff_rate
estimated_emissions_kg = total_kwh * emission_factor

UI should show:
- estimated bill cost
- estimated CO2 emissions
- cost trend
- emissions trend
- cost per day
- emissions per day

5. Recommend Loop
- Backend generates energy-saving recommendations.
- Use deterministic recommendation rules first.
- Use MockAIProvider to generate a friendly summary.
- Recommendations should be based on detected patterns.

Examples:
- If evening usage is high, suggest shifting appliance usage to off-peak hours.
- If weekend usage is high, suggest reviewing weekend appliance behavior.
- If spikes are detected, suggest checking high-load appliances.
- If total usage is high, suggest setting a monthly reduction goal.

Each recommendation must include:
- recommendation_id
- title
- category
- reason
- estimated_savings
- estimated_co2_reduction
- difficulty
- priority
- status

Recommendation categories:
- peak_shift
- appliance_efficiency
- standby_reduction
- behavior_change
- tariff_optimization
- monitoring

6. Act Loop
- User can mark recommendations as:
  - planned
  - in_progress
  - completed
  - dismissed
- User can set a monthly kWh reduction goal.
- User can add notes to actions.
- UI should show action checklist and goal progress.

7. Track Loop
- System tracks progress across sessions.
- For MVP, store history in frontend local state.
- Structure code so database storage can be added later.
- Compare current session against previous session.
- Show:
  - kWh change
  - estimated cost change
  - estimated emissions change
  - completed actions
  - goal progress

8. Improve Loop
- System generates a next-cycle improvement plan.
- Compare completed actions with actual usage changes.
- Suggest what to focus on next.
- Mock AI provider should generate a simple next-month energy plan.

Example:
“Your evening peak remained high this month. Next cycle, focus on shifting laundry, charging, and cooking-related loads outside 7 PM to 10 PM.”
